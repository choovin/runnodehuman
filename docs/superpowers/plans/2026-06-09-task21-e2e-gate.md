# Task 21: End-to-End Smoke Test — Status

**Status:** Deferred to CI. Cannot run in this local dev environment.

## Why deferred

Task 21 (per the implementation plan) is to build the full electron-builder
installer on all 3 platforms and verify the 8 runtimes launch from the
bundle. Two blockers prevent this in the current local env:

1. **`runtimeManifest.sha256` is still `"REPLACE_AFTER_FIRST_FETCH"`** —
   the placeholder values were added when the manifest field was first
   created (Task 8, commit `5a21af4`). They get replaced with real
   SHA-256 digests by the first successful `setup-bundled-runtimes`
   fetch, which requires network + ~1.5 GB of downloads. Until that
   happens, the `beforePack` hook `verifyBundledRuntimes` in
   `scripts/electron-builder-hooks.cjs` correctly throws to prevent
   shipping with unverified runtimes.

2. **Cross-platform builds need all 3 OSes** — macOS DMG, Linux AppImage,
   Windows NSIS. Even with sha256 replaced, building Windows NSIS
   requires a Windows runner (or wine); Linux AppImage requires
   `dpkg-tools`/`fakeroot`. Local macOS dev can only exercise the
   darwin-arm64 slice.

## CI gate (to run on first push)

1. CI runner with network access executes
   `node scripts/setup-bundled-runtimes.cjs --all`.
   This populates `vendor/bundled-runtimes/{node,python,git,gh,
   claudecode,codex,hermes,openclaw}/<version>/{darwin-arm64,
   darwin-x64,linux-x64,win32-x64}/` and updates `runtimeManifest.sha256`
   for each `(platform, arch)` slice.
2. CI commits the updated `package.json:runtimeManifest` back to the
   branch.
3. CI runs the 3 build jobs (`predist:mac` + `predist:linux` +
   `predist:win`).
4. `verifyBundledRuntimes` (in `beforePack` hook) checks every runtime
   binary's SHA-256 against the manifest. If any mismatch, the build
   fails.
5. `signBundledRuntimesWindows` (in `afterPack` hook) re-signs all
   `.exe`/`.dll` files under `wesight-runtime/` with the
   authenticode certificate.
6. `notarize.js` re-signs the macOS bundle with `codesign --force
   --deep` and submits to Apple notary.
7. CI artifacts: 3 installers (~1.5-2.5 GB each) attached to the
   workflow run.

## Local dry-run (no downloads)

If you want to verify the chain without downloading, run:

```bash
# Generate vendor/bundled-runtimes/ with synthetic test runtimes
node scripts/setup-bundled-runtimes.cjs --dry-run
# Then attempt a mac build (will fail at verifyBundledRuntimes
# because the synthetic binaries don't match the sha256 placeholders,
# but the failure will tell you which slice is missing)
npm run predist:mac
```

The `--dry-run` flag (not yet implemented — Task 21 follow-up) would
short-circuit all `fetch*` calls and emit empty `.keep` files in
their place. This is a sensible follow-up if local E2E becomes a
regular need.
